# 表达式求值算法

本文档介绍四种与表达式求值相关的经典栈算法，并附对应的 C 实现。

---

## 1. 后缀表达式求值（Postfix Evaluation）

**文件：** `postfix_evalate.c`

### 算法思路

后缀表达式（逆波兰表达式）中，运算符紧跟在其两个操作数之后，因此不需要括号也不需要考虑优先级。求值只需一个操作数栈：

1. 从左到右扫描 token。
2. 若是数字，压入操作数栈。
3. 若是运算符，从栈中弹出两个操作数 `a`（先弹）和 `b`（后弹），计算 `b op a`，将结果压回栈。
4. 扫描结束后，栈顶即为最终结果。

### 示例

```
输入：56 20 - 4 2 + /

步骤：
  56        → 栈: [56]
  20        → 栈: [56, 20]
  -         → 弹出 20, 56；计算 56-20=36 → 栈: [36]
  4         → 栈: [36, 4]
  2         → 栈: [36, 4, 2]
  +         → 弹出 2, 4；计算 4+2=6 → 栈: [36, 6]
  /         → 弹出 6, 36；计算 36/6=6 → 栈: [6]

结果：6
```

### 关键代码片段

```c
if (strcmp(token, "+") == 0) {
    double a, b;
    stack_pop(stack, &a);   // 后弹的是右操作数
    stack_pop(stack, &b);   // 先弹的是左操作数
    stack_push(stack, b + a);
}
```

**时间复杂度：** O(n)，每个 token 恰好入栈/出栈一次。

## 2. 全括号中缀表达式求值（Fully Parenthesized Infix Evaluation）

**文件：** `fully_parenthesized.c`

### 算法思路

全括号表达式保证每个运算符都被一对括号包围，例如 `( 1 + ( ( 2 + 3 ) * ( 4 * 5 ) ) )`。这使得括号本身就编码了运算顺序，无需考虑优先级。算法使用两个栈：操作数栈和运算符栈。

1. 遇到 `(`：忽略（仅作为结构标记）。
2. 遇到数字：压入操作数栈。
3. 遇到运算符 `+` `-` `*` `/`：压入运算符栈。
4. 遇到 `)`：从运算符栈弹出一个运算符，从操作数栈弹出两个操作数，计算结果后压回操作数栈。
5. 扫描结束后，操作数栈顶即为结果。

### 示例

```
输入：( 1 + ( ( 2 + 3 ) * ( 4 * 5 ) ) )

遇到 )（2 + 3）：弹出 +，弹出 3 和 2，计算 2+3=5，压入 5
遇到 )（4 * 5）：弹出 *，弹出 5 和 4，计算 4*5=20，压入 20
遇到 )（5 * 20）：弹出 *，弹出 20 和 5，计算 5*20=100，压入 100
遇到 )（1 + 100）：弹出 +，弹出 100 和 1，计算 1+100=101，压入 101

结果：101
```

### 关键代码片段

```c
} else if (strcmp(token, ")") == 0) {
    char op = opstack_pop(ops);
    double a, b;
    stack_pop(values, &a);
    stack_pop(values, &b);
    // 计算 b op a 并压回
    stack_push(values, apply(b, op, a));
}
```

**时间复杂度：** O(n)，每个 token 恰好处理一次。

## 3. 一般中缀表达式求值（General Infix Evaluation）

**文件：** `infix_evaluate.c`

### 算法思路

一般中缀表达式不要求全括号，需要正确处理运算符优先级（`*` `/` 高于 `+` `-`）和左结合性。算法同样使用操作数栈和运算符栈，核心在于：**遇到新运算符时，先将栈中所有优先级 ≥ 当前运算符的运算符弹出并计算，再压入当前运算符**。

完整规则：

1. 遇到数字：压入操作数栈。
2. 遇到 `(`：压入运算符栈（作为哨兵，优先级最低）。
3. 遇到运算符 `op`：
   - 循环：若运算符栈非空、栈顶不是 `(`、且栈顶优先级 ≥ `op` 的优先级，则弹出栈顶运算符并计算。
   - 将 `op` 压入运算符栈。
4. 遇到 `)`：循环弹出并计算，直到遇到 `(`，然后丢弃 `(`。
5. 扫描结束后，将运算符栈中剩余的运算符依次弹出并计算。
6. 操作数栈顶即为结果。

### 示例

```
输入：3 + 4 * 2

  3         → 值栈: [3]，运算符栈: []
  +         → 运算符栈为空，直接压入 → 运算符栈: [+]
  4         → 值栈: [3, 4]
  *         → 栈顶 + 优先级(1) < * 优先级(2)，不弹出，压入 * → 运算符栈: [+, *]
  2         → 值栈: [3, 4, 2]
  结束       → 弹出 *：4*2=8 → 值栈: [3, 8]
             → 弹出 +：3+8=11 → 值栈: [11]

结果：11
```

```
输入：( 3 + 4 ) * 2

  (         → 运算符栈: [(]
  3         → 值栈: [3]
  +         → 栈顶是 (，不弹出，压入 + → 运算符栈: [(, +]
  4         → 值栈: [3, 4]
  )         → 弹出 +：3+4=7 → 值栈: [7]；弹出并丢弃 (
  *         → 运算符栈为空，压入 * → 运算符栈: [*]
  2         → 值栈: [7, 2]
  结束       → 弹出 *：7*2=14 → 值栈: [14]

结果：14
```

### 关键代码片段

```c
// 遇到运算符时，先弹出优先级更高或相等的运算符
while (!opstack_is_empty(ops) &&
       opstack_peek(ops) != '(' &&
       precedence(opstack_peek(ops)) >= precedence(token[0])) {
    apply_op(values, opstack_pop(ops));
}
opstack_push(ops, token[0]);
```

**时间复杂度：** O(n)，每个运算符最多入栈/出栈一次。

## 4. 中缀转后缀（Infix to Postfix，调度场算法）

**文件：** `infix_to_postfix.c`

### 算法思路

调度场算法（Shunting-yard Algorithm，由 Dijkstra 提出）将中缀表达式转换为后缀表达式，转换规则与算法 3 几乎相同，区别在于：**运算符不是立即计算，而是输出到结果字符串**。

> [!TIP]
> 操作数的次序不变。

规则：

1. 遇到数字：直接输出到结果。
2. 遇到 `(`：压入运算符栈。
3. 遇到运算符 `op`：
   - 循环：若栈顶不是 `(` 且栈顶优先级 ≥ `op`，则弹出栈顶运算符并输出。
   - 将 `op` 压入运算符栈。
4. 遇到 `)`：循环弹出并输出，直到遇到 `(`，然后丢弃 `(`。
5. 扫描结束后，将运算符栈中剩余运算符依次弹出并输出。

### 示例

```
输入：3 + 4 * 2

  3   → 输出: "3"
  +   → 栈: [+]
  4   → 输出: "3 4"
  *   → + 优先级 < *，不弹出，压入 → 栈: [+, *]
  2   → 输出: "3 4 2"
  结束 → 弹出 *，输出: "3 4 2 *"
       → 弹出 +，输出: "3 4 2 * +"

结果：3 4 2 * +
```

```
输入：1 + ( ( 2 + 3 ) * ( 4 * 5 ) )

结果：1 2 3 + 4 5 * * +
```

### 关键代码片段

```c
// 遇到运算符：先将高优先级运算符输出，再压栈
while (!opstack_is_empty(ops) &&
       opstack_peek(ops) != '(' &&
       precedence(opstack_peek(ops)) >= precedence(token[0])) {
    char op[3] = {opstack_pop(ops), '\0'};
    strcat(out, " ");
    strcat(out, op);
}
opstack_push(ops, token[0]);
```

**时间复杂度：** O(n)，每个 token 恰好处理一次。

## 算法对比

| 算法 | 栈数量 | 括号要求 | 优先级处理 | 输出 |
|------|--------|----------|------------|------|
| 后缀求值 | 1（操作数） | 无需括号 | 无需（后缀已编码） | 数值结果 |
| 全括号中缀求值 | 2（操作数 + 运算符） | 必须全括号 | 无需（括号已编码） | 数值结果 |
| 一般中缀求值 | 2（操作数 + 运算符） | 可选括号 | 需要（优先级比较） | 数值结果 |
| 中缀转后缀 | 1（运算符） | 可选括号 | 需要（优先级比较） | 后缀字符串 |

算法 3 和算法 4 的核心逻辑完全相同（均为调度场算法），区别仅在于：算法 3 遇到可弹出的运算符时立即计算，算法 4 则将其输出到结果串。
